Methodology

Asa applies four complementary forensic tests ("fangs") to each study in your dataset. Each fang probes a different dimension of data plausibility — from granularity of means to distributional properties of raw digits. Studies are classified by a severity-weighted composite verdict.

Workflow
1. Data Entry → parse CSV / manual rows (mean, SD, N, p-value, test stat)
2. The Gauntlet → run all 4 fangs per study, compute weighted verdict
3. Serpent's Eye → study × fang heatmap for pattern detection
4. Forensic Reports → per-study detail with plain-language interpretation
5. Export → CSV, JSON, or TruthCert receipt bundle

Verdict scoring

Each fang returns PASS, WARN, FAIL, or SKIP (insufficient data). The composite verdict uses severity weights:

Result Weight Meaning
FAIL 2 Mathematical impossibility or decision-altering error
WARN 1 Minor discrepancy within rounding tolerance
PASS 0 Consistent with reported values
SKIP — Insufficient data for this test

Overall: Suspicious = any FAIL (score ≥ 2); Caution = WARNs only (score 1); Clean = all PASS/SKIP.

Fang 1: GRIM (Granularity-Related Inconsistency of Means)

GRIM tests whether a reported mean is mathematically possible given the sample size and number of items. Integer-scale data (e.g., Likert scales) can only produce means at specific granular steps — if the reported mean falls between these steps, the data cannot be real.

Applicability: Discrete/count data only. Skipped for continuous measures.

Brown, N. J. L., & Heathers, J. A. J. (2017). The GRIM test: A simple technique detects numerous anomalies in the reporting of results in psychology. Social Psychological and Personality Science, 8(4), 363–369.

Fang 2: SPRITE (Sample Parameter Reconstruction via Iterative Techniques)

SPRITE attempts to reconstruct plausible individual-level data that could produce the reported mean and SD. If no valid dataset can be found within bounded iterations, the reported statistics are flagged as potentially impossible for the given scale constraints.

Applicability: Requires mean, SD, N, and scale bounds. Skipped if scale max is unset.

Heathers, J. A. J., Anaya, J., van der Zee, T., & Brown, N. J. L. (2018). Recovering data from summary statistics: Sample Parameter Reconstruction via Iterative TEchniques (SPRITE). PeerJ Preprints, 6, e26968v1.

Fang 3: statcheck

statcheck recomputes p-values from reported test statistics (t, F, χ², z, r) and compares them against reported p-values. Discrepancies beyond rounding tolerance indicate reporting errors; gross discrepancies that change statistical significance are flagged as decision errors.

Applicability: Requires test statistic + df + reported p-value. Supports one-tailed and inequality formats.

Epskamp, S., & Nuijten, M. B. (2016). statcheck: Extract statistics from articles and recompute p values. R package version 1.3.0.

Fang 4: Terminal Digit Analysis (TDA)

TDA examines the distribution of last digits across all numeric values in the dataset. In authentic data, terminal digits are approximately uniformly distributed (each digit 0–9 appears ~10% of the time). Fabricated data often shows non-uniform patterns, detectable via a χ² goodness-of-fit test.

Applicability: Requires ≥20 numeric values for reliable χ² (df=9). Fewer values produce a SKIP.

Mosimann, J. E., Wiseman, C. V., & Edelman, R. E. (1995). Data fabrication: Can people generate random digits? Accountability in Research, 4(1), 31–46.

Technical Details

Math library: Pure JavaScript implementations of the standard normal CDF (Abramowitz & Stegun 26.2.17), regularized incomplete beta function (Lentz continued fraction from Numerical Recipes), and derived t/F/χ² CDFs. Validated against R's pnorm, pt, pf, pchisq to <0.001 tolerance.

SPRITE engine: Seeded xoshiro128** PRNG for deterministic reconstruction (default 10,000 iterations). Uses swap-based optimization starting from a uniform initialization.

TruthCert: Export bundles include a SHA-256 content hash over all input data + results, ensuring tamper-evident provenance. The hash covers raw inputs, settings, and every fang result.

Privacy: All computations run entirely in-browser. No data is transmitted to any server. localStorage is used only for user preferences (dark mode, default settings).

Disclaimer

Asa flags statistical anomalies for investigation, not as proof of fraud. GRIM and SPRITE failures prove mathematical impossibility of reported statistics — but the cause could be transcription error, rounding differences, or unit confusion rather than fabrication. statcheck flags may arise from unconventional rounding or one-tailed tests. TDA deviations may reflect legitimate data patterns. Always investigate flagged studies in context before drawing conclusions about data integrity.
